> AI models require relevant and dynamic information to provide the most accurate and helpful responses. Msty introduces powerful **Push** and **Pull** context modes, designed to efficiently integrate external data into your AI's conversational flow. These modes work seamlessly with features like **Knowledge Stacks**, **Real-Time Data (RTD)**, and **Live Contexts**, empowering your AI with the right information at the right time.

## Introduction to Context Modes

In [Msty](https://msty.ai), managing context is crucial for intelligent AI interactions. Whether it's fetching the latest news, pulling specific data from an external API, or referencing internal knowledge, the way this information is provided to the AI model directly impacts its performance and the quality of its responses.

This guide will explain the two primary methods for context management: **Push Mode** and **Pull Mode**, detailing how they work, when to use them, and how they integrate with Msty's advanced features.

## Understanding Push Mode

**Push Mode** is designed for proactive and automatic data injection. In this mode, relevant information is fetched and provided to the AI model *before* it even begins to formulate its response.

### How Push Mode Works

1.  **Query Synthesis**: Based on your initial prompt or query, Msty (using the same model) can synthesize one or more search queries.
2.  **Automatic Data Fetch**: These synthesized queries are then parsed to your chosen Real-Time Data (RTD) source (e.g., SearXNG, a specific API endpoint, or an internal Knowledge Stack).
3.  **Context Injection**: The search engine or data source returns results, which are then **inserted directly as context** into the AI model's input before the model attempts to answer your original query.

> **Push Mode**: fetches data automatically; results appear and are added to the model’s context.

### Key Characteristics of Push Mode

*   **Proactive**: Data is provided automatically and immediately.
*   **Pre-emptive**: The model receives necessary information upfront.
*   **Broad Context**: Ideal for providing initial, foundational context for a query.

### Example Scenario (Push)

Imagine you ask: "What are the pros and cons of Msty AI?"
In Push Mode, Msty might automatically generate a search query like "msty ai good and bad things," run it through a search engine, and then push the collected results (pros and cons) into the model's context. The model then uses this pre-loaded information to construct its answer.

## Understanding Pull Mode

**Pull Mode** offers a more dynamic and model-driven approach to data retrieval. In this mode, the AI model *actively decides* when and what data it needs to fetch, utilizing a set of available tools.

### How Pull Mode Works

1.  **Tool Awareness**: The AI model is aware of a list of available "tools" (e.g., a web search engine, a weather API, a specific data endpoint).
2.  **Model-Driven Retrieval**: When the model determines it needs specific information to answer a question or proceed with a task, it **calls one of these tools**.
3.  **Dynamic Parameters**: The model fills in any necessary arguments or dynamic parameters required by the tool (e.g., a city name for a weather tool, a query for a search engine).
4.  **Contextual Integration**: The tool executes, and its results are then pulled back and **persisted and visible alongside push-mode data** in the model's context.

> **Pull Mode**: Model sees tools in the tool list; it calls them when needed, with arguments filling placeholders. Results are persisted and visible alongside push-mode data.

### Key Characteristics of Pull Mode

*   **Reactive**: Data is fetched *only when needed* by the model.
*   **Intelligent**: The model makes intelligent decisions about what information to retrieve.
*   **Focused**: Allows for deeper, more targeted investigation.

### Example Scenario In Live Context (Pull)

User: "What's the weather like in New York today?"

1.  The model identifies "New York" as a dynamic parameter for a weather tool.
2.  The model calls the weather tool, providing "New York" as the city parameter.
    *   **Tool Example**: `https://wttr.in/{city:"enter city name"}?format=j1`
3.  The tool executes, fetches the weather data for New York, and the results are pulled into the model's context.
4.  The model then uses this specific, real-time weather data to answer the user's query.

### Advanced Pull: Deepening the Search(RTD)

A significant advantage of Pull Mode is the model's ability to **call a Real-Time Data (RTD) tool again** as a function. This allows the model to search deeper on a discovered piece of context that it wants to fine-tune after an initial search. This iterative querying capability enables more nuanced and precise information gathering.

## Integration with Msty Features

Both Push and Pull modes are fundamental to how Msty interacts with external data sources and knowledge bases. They are available and accessible across several key features:

*   **Knowledge Stacks**: For accessing structured internal data.
*   **Real-Time Data (RTD)**: For fetching live information from external services or search engines.
*   **Live Contexts**: For dynamically integrating data from specified endpoints.

Conclusion
Msty's Push and Pull context modes provide a flexible and powerful framework for managing information flow to your AI models.

Push Mode ensures immediate, broad context.
Pull Mode enables intelligent, on-demand data retrieval and deeper investigation.
By leveraging these modes, along with Msty's Knowledge Stacks, Real-Time Data, and Live Contexts, you can empower your AI with highly relevant, dynamic, and accurate information, leading to richer conversations and more precise responses.
