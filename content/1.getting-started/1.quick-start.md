Quick Start is the fastest path for new users to go from install to first chat.

## Choose Your Path

- **Desktop (recommended):** Best path for most new users and supports local model workflows.
- **Web (Aurum):** Use Msty Studio in the browser with remote providers or a Desktop/Sidecar connection.

## Msty Studio Desktop Quick Start (Recommended)

::steps

### Install and open Msty Studio
[Download Msty Studio Desktop](/getting-started/download), install it, and open the app.

### Choose how you plan to use Msty Studio
Select **Just exploring**, **For work**, or **I'm a developer**. Msty uses this choice to tailor the recommended model and optional starter resources.

### Connect your first model
Choose either:

- **Recommended local model:** Msty checks your hardware, recommends a compatible local engine and model, and lets you adjust the recommendation from **Light** to **Powerful** before downloading.
- **Online provider:** Connect OpenAI, Claude, xAI, Gemini, OpenRouter, or another [online provider](/managing-models/online-providers). Add at least one model to the provider before continuing.

Local setup displays service and model download progress. You can continue into the app while a local model finishes downloading in the background.

### Choose what to do first
After model setup, choose a starting workflow:

- Open a normal conversation
- Connect an image model and open Media Studio
- Import the Msty Studio Docs Knowledge Stack and create the **Msty Clippy** persona
- Install suggested personas based on your selected intent

### Start working
Msty opens the selected workflow with your model and starter resources ready to use.

::

::alert{type="info" icon="tabler:info-circle"}
Msty stores your data locally on your device. Data does not sync automatically between devices.
::

::alert{type="info" icon="tabler:info-circle"}
Msty Studio can install and manage Ollama, MLX, or Llama.cpp services for local models. The recommended engine depends on your operating system and hardware.
::

### Optional: Install Local AI Services From an Archive File

If the standard service installer is not suitable for your system, use the service settings to install a compatible local archive.

![import ollama archive](/images/quick-start/import-ollama-archive.png)

For Ollama:

1. Download the correct archive for your OS from [Ollama Releases](https://github.com/ollama/ollama/releases){target="_blank"}.
2. Open **Settings > Local AI > Service Version** and choose the archive file option.
3. Select the downloaded archive file.

::alert{type="info" icon="tabler:info-circle"}
If Local AI setup fails with the default installer, archive install is a useful fallback.
::

## Msty Studio Web Quick Start :aurum-tag

::steps

1. Purchase an [Aurum license](https://msty.ai/pricing){target="_blank"} and keep your license key handy.
2. Open [https://msty.studio](https://msty.studio).
3. Enter your license key and click **Next**.
4. Add a model provider:
   - Add a remote provider (OpenAI, Anthropic, Gemini, and others)
   - Or connect Studio Web to [Msty Studio Desktop](/settings/remote-connections) or [Msty Sidecar](/how-tos/sidecar)
5. Finish setup, choose a starting workflow, and run your first prompt.

::

::alert{type="info" icon="tabler:info-circle"}
**Safari user?** Safari is not directly supported by Msty Studio. You can still connect through [Remote Connections](/settings/remote-connections) using tunneling.
::
