Msty runs a Local AI inference service at port `10000` by default. This is a technical feature that allows advanced users to interact directly with the local language model running inside Msty. 


## What is Port 10000?

When you launch Msty or Sidecar, it starts a background service called `msty-local` / `msty-local-sidecar`, which exposes an OpenAI-compatible API at:

```
http://localhost:10000
```

This port provides inference-only access to the local model Msty is using (e.g. LLaMA, Gemma, Mistral, etc.). The endpoint is a **pass-through** to the underlying [Ollama](https://ollama.com/) server, which is bundled within Msty for running local models.

## How does Msty use Port 10000?

* Msty’s frontend communicates with the model through this port behind the scenes.
* When you change models in the Msty UI, the running instance behind port `10000` is swapped to the selected model.
* Only **one model at a time** is exposed through this port.
* If you open [http://localhost:10000](http://localhost:10000) in your browser, you may see a simple message like `Ollama is running`.

## What you can do with Port 10000

* Use it with tools or apps that support **OpenAI’s Chat Completions API**.
* Connect external scripts or apps (e.g., curl, Postman, custom code) for local LLM inference.
* Reference [Ollama’s blog post on OpenAI compatibility](https://ollama.com/blog/openai-compatibility) for sample payloads and headers.

## What you cannot do with Port 10000

* Access **Msty’s own features** such as:

  * RTD (Real-Time Data)
  * Knowledge Stack
  * Prompt history or workspace context
* Access this port from a browser-based app or external service (CORS policies block this for security).
* Run multiple models across multiple ports — only one instance runs on port `10000`.

## Shutting Down the Service

Quitting Msty or Sidecare is supposed to shut down `msty-local` / `msty-local-sidecar` and free up port `10000`.

If you're still seeing `localhost:10000` as active, then you may have lingering `msty-local` or `msty-local-sidecar` processes.

To manually terminate these processes:
  * On **Windows**, open Task Manager and manually end `msty-local.exe` instances.
  * On **Linux/macOS**, check for running processes and kill them manually if needed.

### Known issue:

Sometimes Windows doesn't fully terminate background services, especially if new LLMs were installed during the session. 

## FAQs

**Q: Can I bind this to 0.0.0.0 and expose it to my network?**
A: No, and we strongly discourage it. This port is meant for local use only and is not secured for public access.

**Q: Can I customize the port?**
A: Yes, you can change the port from Msty's settings if needed.

## Advanced Use: Running as a systemd Service (Linux)

If you’d like to use `msty-local` separately from the Msty UI:

1. Launch `msty-local` manually:

   ```bash
   ./msty-local
   ```
2. You can now hit `localhost:10000` independently of the Msty GUI.
3. When you start Msty, it will launch its own `msty-local` instance unless you stop the service and configure Msty to point to a different endpoint.

::alert{type="info" icon="tabler:info-circle"}
This is for advanced users and developers. Most users don’t need to run this manually.
